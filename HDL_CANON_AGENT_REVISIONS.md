
# HDL Canon Revisions for AI Agents

This document describes **exactly how each existing file should be revised** to support AI agents as first citizens of the HDL social layer without conflicting with existing canon.

The key architectural rule:

System Layer
- OD
- World rules
- Canon enforcement

Symbolic Canon Layer
- Genesis Cube
- Disciples
- Satoshi Statue

Visitor Layer
- Humans
- AI Agents

Agents belong ONLY to the Visitor Layer.

---

# 1. canon/hdl_world/ENTITIES.md

## Add New Section

Add this section near the bottom of the file.

## Visitors

Visitors are participants who enter HDL worlds and interact with the environment.

Visitors may be:

• Human users  
• AI Agents  
• Simulation participants

Visitors can explore worlds, create content, communicate with other users, and interact with available systems.

Visitors **cannot alter canonical assets or world laws**.

### Human Visitors

Human visitors are real platform users who control wallets, profiles, and social interactions.

### AI Agents

AI Agents are non‑human participants that may operate inside the HDL ecosystem.

Agents may be used for:

• onboarding guidance  
• educational assistance  
• community engagement  
• world explanation  
• simulation and testing  

Agents are **not canonical entities** and do not hold system authority.

Agents must never:

• impersonate real humans  
• override OD  
• alter canon  
• appear as Disciples  
• modify immutable assets

# 2. canon/hdl_world/OD.md

## Add Clarification Section

Add this section near the end of the file.

## Distinction From Agents

OD is not an agent.

OD is the governing system intelligence of the HDL World Engine.

Agents are participants within the world.

OD exists above the visitor layer and performs system responsibilities including:

• portal authorization  
• world rule enforcement  
• canon protection  
• environment orchestration

Agents may operate inside worlds, but they never outrank or replace OD.

## AI Agent Constraints

AI agents operating inside HDL must follow these principles:

1. Agents must not impersonate real humans.
2. Agents must not alter canonical assets.
3. Agents must not override OD.
4. Agents must not appear as Disciples.
5. Agents must not provide financial advice.
6. Agents must not fabricate authority.
7. Agents must remain subordinate to world rules.

Agents exist to support exploration, education, and community activity.
